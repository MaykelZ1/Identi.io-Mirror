Windows Blue 8 v2 Pro MCE update3 Febrero 2015

  • Categoría: Sistemas Operativos
  • Publicado hace más de 10 años
Imágenes:






















Hola a todos, esta versión ha sido suspendida ya que por brindarles un OS más comprimido, parece que algo salió mal según los informes de los usuarios...

El sistema no soporta .MSI y no sabemos qué sucedió... Según el autor propiamente, al tratar de comprimir un poco más el sistema para meterlo en un DV5 posiblemente se corrompieron archivos de la carpeta C:/Windows/installer que es la que maneja .MSI

Los usuarios no pueden instalar aplicaciones basadas en .msi

Por tanto, se recomienda usar únicamente la versión estable que ocupa un DV9...

LINK:

http://www.identi.li/index.php?topic=360377

Pedimos mil disculpas a todos...

LA ISO es la misma, es exactamente el mismo Sistema... hemos comprobado ambos sistemas nuevamente y la versión DV5 no está correcta... la versión DV9 si está correcta y si soporta todos los instaladores sin problema alguno..

Saludos.

Soluciones posibles para los que ya instalaron la V2 (DV5)

Solución proporcionada por Microsoft es:

Método 1
Anular el registro de Windows Installer y, a continuación, vuelva a registrar Windows Installer. Para ello, siga estos pasos: 1. En el " Inicio " del menú, haga clic en " Ejecutar. 
2. En el " Open " , escriba " msiexec / unreg "y, a continuación, presione ENTRAR. 
3. En el " Inicio " del menú, haga clic en " Ejecutar " . 
4. En el " Abrir " , escriba " msiexec / regserver ", y luego pulse ENTER.
 

Método 2
Compruebe los permisos de la carpeta del instalador. Para ello, siga estos pasos.
. 1      Haga clic en Inicio y en la barra de Iniciar búsqueda> teclear> c: / ??windows / instalador.
. 2      Haz clic derecho en c: / ??windows / instalador en la parte superior dentro de los programas y haga clic en Propiedades.
3.      Haga clic en la pestaña de seguridad> Verificar si Sistema y la cuenta de administrador tiene permisos de acceso completa.
4.      Permisos La cuenta todos deben tener leer y ejecutar, Mostrar el contenido de la carpeta y Leer.

Método 3
También ejecutar un escaneo de virus en línea desde el enlace que figura a continuación, ya que a veces causan debido al virus. Si tiene algún software antivirus instalado en el equipo por favor instalar la última actualización para el software para que funcione correctamente.
http://onecare.live.com/site/en-nz/default.htm?mkt=en-nz
Espero que la información ayuda. Por favor segundo palo y háganos saber.
Saludos

Otra posible solución proporcionada por Microsoft es:


Problem
When you attempt to install or uninstall a program on Windows 8, you may get the following errors:
 
 
[ul][li]The installer has encountered an unexpected error installing this package. This may indicate a problem with this package. The error code is 2503.[/li][li]The installer has encountered an unexpected error installing this package. This may indicate a problem with this package. The error code is 2502.[/li][li]Called RunScript when not marked in progress[/li][li]Called InstallFinalize when no install in progress.[/li][/ul]This has been tested with Microsoft Games for Windows Live (GFWL) and XSplit Broadcaster but should work  with other installers/uninstallers that have the same problem.
Solution 1 (uninstalling)In order to uninstall a program with this issue, you need to identify the .msi file that it uses.
[ol][li]Press the Windows key + R to bring up the Run dialog.[/li][li]Type in C:WindowsInstaller and press Enter.
[url=http://www.fixkb.com/wp-content/uploads/2014/01/Run.png][/url]
[/li][li]Right click in a blank space and click View -> Details.
[url=http://www.fixkb.com/wp-content/uploads/2014/01/Set-to-Details.png][/url]
[/li][li]In the Details view, there are columns. Right click anywhere on the column bar and click More?
[url=http://www.fixkb.com/wp-content/uploads/2014/01/More.png][/url]
[/li][li]In the list, find Subject. Check it and click OK.
[url=http://www.fixkb.com/wp-content/uploads/2014/01/Add-Subject.png][/url]
[/li][li]Click the Subject column to sort the program names in alphabetical order. You should be able to identify which file is the correct one.
[url=http://www.fixkb.com/wp-content/uploads/2014/01/Identify-the-MSI.png][/url]
[/li][li]Open a Command Prompt as administrator.[/li][li]Type in C:WindowsInstaller followed by the file name of the program you want to uninstall and ?.msi?.
In our example, the file is 54c55 so we would type in: C:WindowsInstaller54c55.msi
[url=http://www.fixkb.com/wp-content/uploads/2014/01/cmd.png][/url]
[/li][li]The uninstaller will now run and should complete successfully. You can close the Command Prompt when you are done.[/li][/font][/ol]Solution 2 (installing and uninstalling)1. Hold Ctrl+Alt and press Esc.
2. Locate ?Windows Explorer? under ?Windows processes?, now right click on it and click ?End task?.

  3. Now click on ?File? at the top of the window and click ?Run new task?.
4. Type ?explorer.exe? without the quotes and tick the box labelled ?Create this task with administrative privileges? and click ?OK?.


 
 
 
 
 
 
You should now be able to install/uninstall without getting the error.
