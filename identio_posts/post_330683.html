Como Instalar Programas En Ubuntu

  • Categoría: Hazlo tu mismo
  • Publicado hace más de 11 años



Lejos de la creencia de que en Linux el instalar programas es una tarea complicada y sólo apta ''gurús'' de la informática, vamos a ver que, salvo en contadas ocasiones, es un tarea sencilla y rápida en la que ni tan si quiera hace falta reiniciar el sistema. En este tutorial se explica como instalar programas para Ubuntu.
 
Añadir/quitar:
 
Este es el sistema más sencillo, sólo debemos ir a Aplicaciones/añadir quitar y veremos la siguiente ventana:
 

 
Aquí sólo debemos marcar los paquetes que nos interese y dar al botón Aplicar cambios. Para eliminar sería igual.
 
Synaptic:
 
Synaptic es otra herramienta gráfica para instalar paquetes, está en Sistema/administración/gestor de paquetes Synaptic;
 

 
Para instalar un paquete pinchamos en Buscar y escribimos el nombre del paquete que queremos. Una vez localizado lo marcamos y pinchamos en Aplicar. Se puede dar el caso que se marquen por su cuenta otros paquetes necesarios para que funcione el que hemos elegido.
 
Instalando paquetes desde el terminal:
 
1.- Paquetes .deb
 
Aquí es cuando muchos se asustan creyendo que hay que ser un programador para manejarnos con el terminal, pero con aprendernos cuatro comandos veremos que instalar paquetes con este sistema es muy cómodo y rápido.
El formato natural de Ubuntu es el .deb.
 
Primero abrimos un terminal en Aplicaciones/accesorios/terminal y tecleamos:
 
$ sudo aptitude update (este comando actualiza la caché de /etc/apt/sources.list).
$ sudo aptitude install nombre-paquete (nos instala el paquete que queremos).
 
ejemplo: para los jugones, vamos a instalar un juego del estilo Quake, llamado Openarena.
 
$ sudo aptitude update
$ sudo aptitude install openarena
 
Con esto ya tendríamos el juego instalado en nuestro sistema, así sería con todos los programas.
En lugar de aptitude podemos usar apt-get, pero aptitude nos resuelve los problemas de dependencias.
 
Para desinstalar un programa haríamos
 
$ sudo aptitude remove nombre-paquete
 
Arriba he hecho mención a sources.list, pero... ¿qué es eso? pues es una lista de direcciones de ''depósitos'' donde se encuentra los paquetes, donde cada linea indica lo siguiente.
 
- Un tipo de paquete, deb o deb-scr para Debian y distros derivadas de ella, como Ubuntu.
- Una URL que apunta al archivo donde esta el paquete, CD-ROM, servidor FTP o HTTP
- El nombre de un lanzamiento, que nos permite elegir varias versiones
 
Salvo en casos especiales no es necesario modificar este archivo, pero podemos echarle un vistazo, sin tocar nada, para ver como es tecleando:
 
$ sudo gedit /etc/apt/sources.list (Gedit es simplemente un editor de texto).
 
También es posible bajar paquetes .deb directamente de internet. El sitio recomendable es el oficial http://www.getdeb.net.
 
Una vez descargado, haciendo doble clik sobre él se instalaría.

2.- Paquetes .package
 
Estos paquetes usan una herramienta llamada autopackage que se instala en el sistema con el primer archivo .package que instalemos, el procedimiento es bien sencillo:
 
Una vez descargado el primer paquete .package haremos clik con el botón derecho sobre él y elegimos Propiedades, y en Permisos marcamos la casilla Permitir ejecutar el archivo como un programa. Después haríamos doble clik sobre el archivo y se instalaría.
 

 
Para eliminarlo vamos a Aplicaciones/herramientas del sistema/manage 3rd party sofware, seleccionamos All applications, elegimos la aplicación a eliminar y hacemos click en Remove.
 

 
3.- Paquetes .rpm
 
Los paquetes .rpm son nativos de Red Hat, Fedora...., pero se pueden instalar en Ubuntu, para ello debemos convertir esos paquetes a .deb utilizando la herramienta Alien. Antes de usarla podemos instalarla desde Synaptic como ya he explicado más arriba o tecleando en el terminal:
 
$ sudo aptitude install alien
 
para cambiar el formato del paquete a .deb e instalarlo sería:
 
$sudo alien nombre-paquete.rpm
$sudo aptitude install nombre-paquete
 
Puede darse el caso de que el paquete no sea del todo compatible. Siempre debemos recurrir a paquetes .deb y dejar los .rpm como última opción.
 
4.- paquetes .tar.gz
 
Estos paquetes traen el código fuente y suelen venir acompañados de un archivo readme o install con instruciones de instalación, lo primero es descomprimirlo en nuestra carpeta personal:
 
$ sudo tar xzvf nombre-paquete.tar.gz
 
O haciendo clik con el botón derecho del ratón y dandole a Extraer aquí. Después debemos compilar he instalar el paquete:
 
$ ./configure
$ make
$ sudo make install
 
Y para eliminarlo tecleamos
 
$ sudo make uninstall
 
La instalación de este tipo de paquetes no es aconsejable para los que tienen poca experiencia con Linux, siempre hay alternativas con paquetes .deb
