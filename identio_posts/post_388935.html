Soluciones al problema de Wifi en Windows 8_1








Para todos los que tienen Windows 8.1 y
se les desconecta de Internet el sistema
de manera aleatoria y sin razón alguna
aquí les dejo algunas soluciones que me
han servido al respecto.









La nueva versión de Windows trajo problemas para muchos usuarios, ya que Windows 8.1 no esta detectando correctamente los drivers de algunas tarjetas Wi-Fi o simplemente presenta problemas de ?conexión limitada? en redes que funcionan correctamente. 



PASO 1

Deja de utilizar Windows e instala, por ejemplo, Linux Mint en cualquiera de sus sabores. Lo recomiendo porque a mí me funciona más que bien, pero como saben, hay muchísimas otras alternativas al SO de las ventanas. Lo digo completamente en serio: desde que dejé de utilizar Windows no sólo ha aumentado mi productividad y mi buen humor, sino que además conectarme a mi WiFi sin problemas.

PASO 2

Si eres de los de ?yo nací con Windows y con Windows moriré?, no te preocupes, hay otras posibles soluciones que puedes probar.

- Deshabilitar IPV6
Con la tecla de Windows + Q se abrirá el diálogo de búsqueda. Escribimos ?recursos compartidos? y pulsamos "Enter". Aparece la ventana de "Centro de redes y recursos compartidos", en la que debemos encontrar la opción ?Cambiar la configuración del adaptador?.




Seleccionamos nuestra red, botón derecho y elegimos ?Propiedades?. Solo nos queda desmarcar la casilla de Protocolo de Internet versión 6 (IPv6).




- CAMBIAR EL SERVIDOR DNS

Otras veces, pasa que el sistema NO RESUELVE BIEN los DNS. Esto se resuelve de la siguiente manera:


A) En la parte inferior donde el reloj, seleccionan el icono del Internet.




B) Ya en "centro de redes" xxxx (donde xxxx es tu centro de redes) y entrar a lo seleccionado > Cambiar configuración del adaptador. Buscar la Conexión IPv4 y cambiar el servidor DNS.



- Aquí en la imagen, están los servidores DNS de Google:
8.8.8.8
8.8.4.4

- También puedes usar los predeterminados de Telmex si vives en México:
200.23.242.201
200.23.242.193

- Otra opción es usar los de OpenDNS:
208.67.222.222 
208.67.220.220 
- MODIFICAR COMANDOS DE DNS EN SIMBOLO DE SISTEMA (CMD)

Si hasta aquí sigue dando problemas, deberás armarte con un poco de paciencia y ejecutar algunos comandos con el CMD (símbolo de sistema), ejecutándolo como administrador.





Aquí y dentro del programa de "símbolo de sistema" se deberán ejecutar los siguientes comandos UNO A UNO, copiando el primero, ejecutándolo, copiando el segundo, ejecutándolo y así sucesivamente.


1) netsh int tcp set heuristics disabled

2) netsh int tcp set global autotuninglevel=disabled

3) netsh int tcp set global rss=enabled

4) netsh int tcp show global

El 4) es para comprobar que la práctica en su totalidad de los ajustes están desactivadas.


Después de haber probado todas estas soluciones, pude volver a navegar por Internet con "normalidad" sin que se desconectara de la Internet a cada rato y sin razón aparente.

Espero les sirva este compilado que encontré en varias páginas web que tocan el tema.
