Configurar cisco aironet y cap

  • Categoría: Tutoriales / Cursos
  • Publicado hace más de 11 años
Buenos dias a todos les traigo una aportación de calidad ya que en todo la internet no se exlplica a detalle como configurar nuestro accespoint cisco pues yo me anime a desvelar el secreto. despues de batallar un poco ya que nunca habia configurado un ap cisco vi tutoriales en otros idiomas y les faltaba darme un pequeño gran detalle antes de resetear mi ap nunca decian que habia que montar un ios nuevo , pero bueno aqui explicare paso a paso la configuracion . 
empecemos:

estos son nuestros anfitriones:


este es el arionet 1131
y este es el AIRCAP1602I

ok veremos la forma desde que lo reseteamos  asta que lo configuramos.


primero nuestro accespoint aironet 1131.


este accespoint veremos su diseño de estructura  asi seve cuando se recorre la tapa de proteccion :

bien en el numero 1 tenemos la entrada para el eliminador , 2 es la entrada rj45 para la red, 4 es el puerto rj45 para la consola el 6 botton reset , 7 leds , 8 led de estado del accespoint. (no menciono los otros ya que son piezas que no nos importan por el momento.

Lo que tienes que saber  antes de dar un reset de fabrica a tus cisco es lo siguiente al mometo que lo resetes perdera por asi decirlo su sistema operativo llamado ios .  asi que hay que volverlo a reinstlalar aqui te dire como, entremos en materia.


material:

un cable curzado cross over este lo podemos hacer manualmente con cual quier cable utp.


es esta la configuracion para nuestro cable.

2.-  un cable de consola para entrar en el modo consola de nuestro ap


3.- nuestro sofware terminal para comunicacion con la consola lo podremos descargar de aqui:

https://app.box.com/shared/8vehulvb7u

4.- nuestro sofware 3daemo ftp que servira para inyectar el ios nuevamente a nuestro ap

http://www.firewall.cx/downloads/ftp-tftp-servers-a-clients/16-1-3cdaemon-server-a-client.html


creo que no nos falta nada en cuanto al material.

manos a la obra:

PASO NUMERO 1

abrimos la parte superior de nuestro ap en el caso del aironet 1131 ubicamos nuestro botton reset esta aun lado del puerto de la consola lo mantenemos pulsado y en ese momento conectamos  a la corriente sin soltar el botton reset asta que el ap cisco encienda y cambie el color de los leds en rojo, en ese momento  10 segundos mas seran suficientes para borrar toda la antigua configuracion de nuesto ap. 
* con esta accion queda por si decirlo sin sofware  nuestro cisco y es imposible configurarlo o acceder a el.

PASO NUMERO 2

A.- Actualizacion de Firmware
 
1.- Instalar en pc TFTP server en este caso 3CDaemon
 
2.-Cambiar direccionamiento de pc a 10.0.0.2, mascara 255.255.255.0
 
3.-Ejecutar TFTP y en menu configure TFTP server en la opción Upload/Dowload directory ingresar la ubicación en donde esta el archivo de actualización de Firmware del AP.
 
4.-conectar el AP a la pc con cable cruzado,desconectar  el cable de corriente del AP en seguida encenderlo  manteniendo presionado el botón de Mode hasta que se encienda el led rojo y mantenerlo asi por 10 segundos aprox.
 
5.-Ensegida en el TFTP en la opción de status aparece el mensaje Send of c1130-k9w7-tar.default done,esperar a que termine de hacer la actualización del firmware.

nuesto ios o firmware lo descargaremos de aqui el que es para el caso del aironet 1131

https://mega.co.nz/#!GNZUQCxb!5ENIGbui_iHn1HHsqc-HMpwGhJkfRxoMryOvXk2UhBg

generalmente los firmwares solo se descargan en cisco  para los  empresarios que tiene contrato con ellos asi que les comparto el software. el otro link es para el cisco AIRCAP1602I

https://mega.co.nz/#!XEY0hSwJ!ffs8_9QaPs1_arktFyegW0PbtE-32c9nx1pXvpjCJTY

asta este momento este procedimiento es el mismo para los dos modelos.



esta es la aplicacion del 3daemo ftp  que nos mostrara para indicarle  donde estara nuestro archivo c1130-k9w7-tar.default   (recuerden descomprimir)

B.- Configuracion de AP via consola de valores por default
6.-Una vez terminada la actualización conectarse al AP por consola,el password es Cisco.
 
Ya estando en el prom # entrar a modo configuracion con el comando conf t posteriormente teclear el comando interface BVI 1 y enter.Ya  dentro de la interface ingresar la ip 10.0.0.1 255.255.255.0;guardar los cambios.
aqui utilizaremos nuestro programa que descargamos hyperterminal.

o podemos usar tambien esta configuracion para la consola como se acomoden:
Escribir Enable para ingresar en modo de usuario privilegiado.

           2.      Password por defecto Cisco.

           3.      Escribir configure terminal  para ingresar en modo de configuración.

           4.      Ingresar Interface bvi1 para configurar la interface.

           5.      Ingresar el comando ip address 192.168.1.254 255.255.255.0.

           6.      Salir con el comando exit (2 veces).

           7.      Ingresar write para grabar la configuración.

           8.      Salir de la consola Hyperterminal.

ddespues  quitamos el cable de consola y el croos over y poner nuestro cable normal de ethernet  y seguimos las siguientes instrucciones








C.- Configuracion de AP via web de ip y seguridad
7.-Ingresar al AP via web con el usuario admin y el password Cisco,dentro del menú Security ir al submenú Encription Manager y en el menu Encrytion Modes seleccionar Cipher y en el campo seleccionar AES CCMP,guardar cambios en el botón Apply-Radio 0.
8.-En el mismo menú de Security seleccionar el submenú SSID Manager,en SSID poner el nombre del AP,en interface activar la opción de Radio0-802.11G;en ese mismo menú pero en la opción de Client Authenticated Key Management que se encuentra mas abajo seleccionar la opción de Mandatory en Key Management y habilitar enable WPA,en el campo WPA Preshared Key ingresar la contraseña de AP y salvar configuracion.
9.-Continuando en el mismo menú de SSID Manager desplazarse a la opción de Guest Mode/Infraestructure SSID Settings en la opción de Set Single Guest Mode SSID seleccionar el nombre que se le puso al AP y salvar cambios.
10.-Finalmente en el menú de EXPRESS-SETUP configurar la IP que va a llevar el AP con la mascara y el gw.
 
11.- Accesar el menú Networks Interfaces,en la pestaña de settings seleccionar enable radio para propagar el SSID.
