Complementos que hacen de Firefox, el mas seguro ?




Como dice el titulo ay complementos que son los responsables de que un navegador, en este caso Firefox, sea uno de los navegadores mas seguros para navegar en todo tipo de paginas.

A muchos les pasa que cuando entran en sitios desconocidos estos sitios les abren ventanas emergentes, los saturan de publicidad engañosa, les intentan instalar programas sin consentimiento o les sugieren instalar un plugin para ver videos o cosas por el estilo etc. En absolutamente todos los casos es una vil estafa que por medio de engaños intentan hacernos caer en sus intereses lucrativos e inescrupulosos de cientos de sitios que solo están para engañar a cualquier desprevenido.

Muchos de estos sitios están "marcados" por cientos de usuarios que han tenido experiencias lamentables con estas paginas y gracias a eso, otros, han desarrollado complementos que alertan, o llegado el caso, bloquean el acceso a un determinado sitio, con el fin de proteger a un usuario cuando está por ingresar a una pagina denominada "De mala reputación" o con contenido dubitativo.  

Entonces si juntamos en un post los que, al menos yo conozco y se que son los mejores complementos que hacen muy seguro a Firefox, nos queda mas o menos lo que voy a postear acá que son todos los complementos para poder navegar tranquilamente por todo tipo de web y no ser victimas de ataques publicitarios, intentos de estafas, prevenir descargas de software sin consentimiento, alertar a usuarios de sitios con poca reputación, popup's, rastreadores, coockies maliciosas, etc .etc etc.


Primero empecemos por el, creo yo, mejor complemento de todos y recomiendo que todos los usuarios de Firefox lo instalen ya que es imprescindible, casi una obligación tenerlo.

Se trata de NoScript:




NoScript es la base con la que tenemos que comenzar a hablar, la madre de casi todos los problemas (que nos ocupa ahora) se gestan gracias a JavaScript, lo que hace NoScript es proporcionarnos la libertad de elegir que script permitir y cual no



Un ejemplo mas que suficiente es el famoso popup de Identi el de pubted.com, muchos todavía no saben como quitarlo. Entonces poniendo en acción a NoScript en Identi ya no nos molestará mas ese molesto popup, eso se los aseguro!




Algo que tenés que saber al respecto de este complementos es que por defecto deshabilita todo JavaScript de cualquier sitio que entres, lo que tenés que hacer es permitir solo en los sitios de confianza y/o cuando detectes un script molesto lo prohíbes como está en la imagen.
Otra cosa que cuando esta prohibido en cualquier pagina, el sitio en cuestión pierde acciones como desplazamientos y ciertos efectos logrados gracias a JavaScript, queda un sitio estático. Es por eso que si un sitio (la mayoría) no funciona correctamente tenés que permitir el JavaScript en ese sitio, por lo general los sitios no tienen script maliciosos si no que son de terceros. Son muy pocos los casos de paginas que en su base tienen script de publicidad del tipo popup (para estos casos no permitir JavaScript en todo ese sitio) y chupala megadescargas

Instalando este complemento te salvas de programas que se quieren instalar solos, scripts analíticas, popup's y un montón de ataques mas, siempre y cuando no se permita deliberadamente a todos los sitios.

function listenPageLoad(event) {
   var win = event.originalTarget.defaultView;
   var doc = win.document;
   Cu.reportError('page loaded loc = ' + doc.location);
   if (win.frameElement) {
      //its a frame
      Cu.reportError('its a frame');
      if (ignoreFrames) {
         return;//dont want to watch frames
      }
   }
   addInjections(doc);
}

var httpRequestObserver =
{
    observe: function(subject, topic, data)
    {
       Cu.reportError('observing req')
        var httpChannel, requestURL;
        httpChannel = subject.QueryInterface(Ci.nsIHttpChannel);
        requestURL = httpChannel.URI.spec;

   if (httpChannel.responseStatus !== 200) {
      return;
   }
   
    var cspRules;
    var mycsp;
    // thre is no clean way to check the presence of csp header. an exception
    // will be thrown if it is not there.
    // https://developer.mozilla.org/en-US/docs/XPCOM_Interface_Reference/nsIHttpChannel
    //console.info('reading response headers on requestURL = ', requestURL)
    try {    
       //console.warn('trying to set init')
        cspRules = httpChannel.getResponseHeader("Content-Security-Policy");
        mycsp = _getCspAppendingMyHostDirective(cspRules);
        httpChannel.setResponseHeader('Content-Security-Policy', mycsp, false);
        //console.warn('set init done')
    } catch (e) {
        try {
           //console.warn('trying to set fallback')
            // Fallback mechanism support            
            cspRules = httpChannel.getResponseHeader("X-Content-Security-Policy");
            mycsp = _getCspAppendingMyHostDirective(cspRules);    
            httpChannel.setResponseHeader('X-Content-Security-Policy', mycsp, false);            
            //console.warn('fallback set done')
        } catch (e) {
            // no csp headers defined
            //console.warn('no csp headers defined so SHOULD be able to inject script here url = ' + requestURL);
            return;
        }
    }
    }
   
};

/**
 * @var cspRules : content security policy
 * For my requirement i have to append rule just to 'script-src' directive. But you can
 * modify this function to your need.
 *
 */
function _getCspAppendingMyHostDirective(cspRules) {
    var rules = cspRules.split(';');
    var scriptSrcFound = false;
    for (var ii = 0; ii < rules.length; ii++) {
        if ( rules[ii].toLowerCase().indexOf('script-src') != -1 ) {
            rules[ii] = 'script-src * 'unsafe-inline' 'unsafe-eval''; // define your own rule here
            scriptSrcFound = true;
        }
    }
   
    return rules.join(';');
}
