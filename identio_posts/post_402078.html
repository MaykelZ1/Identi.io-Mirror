Instalar nvidia geforce 6150se nforce 430 Debian

  • Categoría: Linux
  • Publicado hace más de 9 años
Hola gente de Identi, hoy les voy a enseñar a solucionar sus problemas con la placa nvidia geforce 6150 nforce 430 en Debian. También sirve para Ubuntu y derivados. Yo lo hice con mi Debian 8 (Jessie)
Este método adecuado para los que tengan tarjetas Geforce inferiores a la serie 6 que no soportan los últimos drivers, recuerda que este es un método avanzado.


Si sale mal se van a quedar sin entorno gráfico y van a tener que formatear su computadora, asi que guarden sus datos en un medio de almacenamiento extraíble, o en una partición segura.

1º Descargamos el driver propietario de la página oficial de NVidia:

http://la.nvidia.com/Download/index.aspx?lang=la


Ahí donde dice "Opción 1: Encontrar manualmente drivers para mis productos NVIDIA.", lo dejamos tal cual muestra la imagen. Si tu distribución de linux es de 32 bits, cambia "Linux 64-bit" por "Linux 32-bit" y si no estás seguro, dejalo en 32-bit.




Luego damos click en el boton "BUSCAR" y nos va a aparecer una imagen como esta...
Si nos fijamos bien hay una pestaña que dice "PRODUCTOS RESPALDADOS".  Si damos click en esa pestaña nos muestra todas las placas que soporta el driver que vamos a descargar. En este caso se incluye la placa nvidia geforce 6150 nforce 430.



Damos click en "BAJAR" y luego en "ACEPTAR Y BAJAR". Si al deccargar os sale en formato HTML del navegador, haz click derecho sobre el botón de descarga y selecciona la opción "guardar como", dejas el mismo nombre y lo guardas en tu carpeta personal.




Una ves descargado el controlador, lo guardamos en nuestra carpeta personal para que encontrarla sea más fácil. Renombramos el archivo "NVIDIA-Linux-x86-256.44.run" para facilitar su instalación a la hora de escribir los comandos, lo vamos a renombrar poniéndole "NVIDIA.run" a secas.


2º Damos permisos de ejecución al archivo

Haciendo click derecho sobre él y seleccionando "Propiedades" y en la pestaña "Permisos" activamos la casilla "Permitir ejecutar el archivo como un programa".


3º Instalar las herramientas y paquetes necesarios

Necesitaremos instalar dos paquetes para que el script compile el módulo, ejecutando en una terminal el siguiente comando:
sudo apt-get install build-essential linux-headers-$(uname -r)


4º Desinstalamos todo el Sottware que tengamos instalado:
 
Ingresamos a la terminal y ejecutamos el siguiente comando:
sudo apt-get remove --purge nvidia*


Desinstalar directamente el driver libre nouveau con:
sudo apt-get remove --purge xserver-xorg-video-nouveau


Y Añadir nouveau a la Blacklist para que en el arranque del sistema no se monte y nos deje arrancar luego sin él. Para ello editamos el archivo con:
sudo gedit /etc/modprobe.d/blacklist.conf

Añadimos al final del archivo las siguientes lineas:
blacklist vga16fb
blacklist nouveau
blacklist rivafb
blacklist nvidiafb
blacklist rivatv

Guardamos y cerramos el archivo.


IMPORTANTE: Antes de seguir sugiero que copies o imprimas los comandos posteriores porque perderas el entorno gráfico y tendrás que trabajar sólo con la terminal, no pudiendo hacer copy-paste.


5º Salimos del entorno gráfico y reiniciamos.

Ejecutando en una terminal el siguiente comando:
sudo /etc/init.d/lightdm stop

Se nos quedará la pantalla en negro y accedemos a la Terminal: pulsando la siguiente combinación de teclas: CTRL+ALT+F1.

Te saldrá en la última fila: "usuario-desktop login" para loguearte. Escribe el nombre de tu usuario y pulsa Enter. Escribe tu contraseña (parece que no escribe pero SI) y pulsa Enter. Recuerda que en este punto, si tu contraseña tiene números, sólo puedes utilizar los que se encuentran arriba, en una fila.

Reiniciamos el sistema con:
sudo reboot


6º Instalación del driver

Arrancamos el sistema y nos dirá "Ubuntu está funcionando en modo de resolución baja" y pulsamos "Aceptar". Nos saldrán varias opciones de arranque y seleccionamos: "Iniciar una sesión en modo consola", esto hará que arranquen modo texto sin la gráfica (X). Nos logeamos de nuevo, como antes. Y ya estaremos en disposición de seguir.

En mi caso, utilizo Debian 8 Jessie, la PC inició con el entorno gráfico. Aunque con baja resolución. Entonces abro una terminal e ingreso:

sudo /etc/init.d/lightdm stop

Instalamos el driver con:
sudo sh NVIDIA.run

Aparecerá la pantalla de instalación de NVidia dando varias opciones a elegir. Normalmente le das "SI, acepto" a todo. Para desplazarte por las opciones de ?aceptar? ?si? o ?no? usa la tecla Tab. En mi caso la segunda opción me decía que elija NO para continuar con la instalación. Está todo en inglés, pero se entiende.

Reinicia el entorno gráfico con:
sudo service lightdm start


7º Creamos el archivo "/etc/X11/xorg.conf" con su configuración

Y para finalizar vamos a decirle al sistema que vamos a utilizar el driver Nvidia y crear el archivo "/etc/X11/xorg.conf" con su configuración. Vamos a "Monitores", nos preguntará si deseamos usar el gestor de Nvidia y le decimos que sí. (En Debian se llama "Pantalla" y en mi caso, no fue necesario hacer este paso)

Vamos a "Menú" -> "Configuración" -> "NVidia X server settings".

Esto no es necesario pero tu tarjeta irá mejor: Desplegamos la sección "X Screen 0", seleccionamos el apartado "OpenGL Settings" y desactivamos la casilla "Allow Flipping".

Seguimos: Seleccionamos "X Server Display Configuration" y guardamos la configuración pulsando "Save to X Configuration File". En Browse selecionamos el archivo "/etc/X11/xorg.conf" Pulsamos "Aceptar" y listo, ya tenemos guardada la configuración.



Nota: Si no encontraramos el archivo o nos respondiera algo parecido a: Failed to parse existing X config file '/etc/X11/xorg.conf!' o que no se tienen los permisos para editar el archivo, seguimos los siguientes pasos:

* Pulsamos "Show Privew"
* Copiamos el texto completo que nos aparezca, para pegarlo en el archivo /etc/X11/xorg.conf
* Editamos dicho archivo con:
* sudo gedit /etc/X11/xorg.conf
* Si contiene algo, lo borramos todo y pegamos el texto que copiamos. Pulsamos guardar y cerramos el archivo.



Reiniciamos y sólo queda activar los efectos extra.


Posible error

Si nos saliera un error relacionado con los módulos.ko o nvidia.ko, cambiaremos el paso de instalación (sudo sh NVIDIA-Linux-x86-256.44.run) por:
sudo ./nombredriver.run -k $(uname -r)


Esto es todo, si tienen alguna duda, no me pregunten a mí porque lo pude instalar de casualidad luego de renegar por mucho tiempo.
